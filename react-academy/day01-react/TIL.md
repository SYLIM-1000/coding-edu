## 목표

- react가 필요한 이유를 이해하고, react 학숩에 필요한 개발 환경을 구성할 수 있다
- react가 무엇인지 설명할 수 있다
- react를 배우는 이유를 설명할 수 있다
  1. 화면을 재사용할 수 있다. 붕어빵 틀을 만들어 계속 찍어낼 수 있다.
  2. 기능을 수정하기 쉽다. 기능과 화면을 나누어서 관리하기 때문.
  3. 서비스 확장에 유리.
  4. AI 와 함께 개발하기 좋다.
- node.js와 브라우저 JS의 차이를 이해할 수 있다
- npm및 nvm의 역할을 설명할 수 있따
- vite를 이용하여 react 프로젝트를 생성할 수 있다
- 프로젝트를 실행하고 깃허브 저장소와 연결할 수 있다

## react 프로젝트 시작 방법

- node.js = 개발 도구를 움직이는 엔진
- npm 사용 = 필요한 도구(패키지)를 가져오는 앱스토어
- 필요한 패키지 설치
- vite 실행 = React 작업실을 만들어주는 도구

## NPM 명령어 방법

'$ npm install bootstrap'

## NPM 명령어 방법

- 설치 엽 확인
- 역할 이해
- 필요할 때 버전 변경

## package.json

- 다양한 설정을 기록하는 파일.
- 프로젝트 이름, 프로젝트 버전, 실행 명령어, 설치된 패키지, 개발 환경 정보

## package-lock.json

- 설치한 패키지의 정확한 버전 정보를 기록함

## 프로젝트 폴더링 구성

- src: 우리가 가장 많이 작업하는 곳
- public: 이미지 등 정적 파일. 파일 그대로 제공하고 싶을 때 사용.
- package.json
- vite.config.js : Vite 설정 파일

### src 폴더 구성

- main.jsx : React 애플리케이션 시작점
- App.jsx : 메인 화면을 구성하는 파일
- App.css : App.jsx와 관련된 스타일
- index.css : 전체 스타일
- assets : react 코드에서 가져와 사용하는 이미지나 리소스를 관리
